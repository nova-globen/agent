## Architectural Decisions Governing Feature Order

- Tier 00 (Foundations) must complete before work in higher tiers begins.
- Within a tier, prefer features that unblock the most downstream work first.
- Blocked features (waiting on an unbuilt dependency) are listed in `docs/plans/backlog.md` under
  **Blocked**.

## Notes

- Each feature has a corresponding spec in `docs/features/NN.feature-name.md`.
- Each feature's work is tracked in a plan folder under `docs/plans/tier-NN_name/NN-slug/`.
- See `docs/plans/README.md` for the plan directory map.
