This file defines how AI coding agents should operate in this repository.

It is **generated by AgentSync** from `.agent/skills/operating-guide/SKILL.md`.
`AGENTS.md`, `CLAUDE.md`, `.gemini/GEMINI.md`, and `.claude/skills/` are generated projections —
**edit canonical files under `.agent/`, never the generated copies**, then run `dotnet agent sync`.

## Token & Context Discipline

The working transcript — tool output plus files read — dominates token use far more than these
instructions do. Keep it small:

- **Summarize noisy command output.** Never let a full build/restore/test log land in the transcript.
  Build quietly: `dotnet build -v q -clp:ErrorsOnly`; filter test output through `tail` or `grep`.
- **Delegate broad reading.** When a task spans more than ~2 docs or needs a wide search, launch a
  sub-agent (`Explore` or `general-purpose`) and work from its conclusion. Build/test gates go through
  the **verifier** agent — it returns a compact pass/fail, not a full log.
- **Read once, scoped.** Read a file once before a batch of edits; do not re-read files you just edited.
- **Don't re-narrate.** A per-increment recap you also write into `PROGRESS_*.md` belongs there — link
  to it rather than repeating the prose in the transcript.

## Mandatory Practices

- Follow `docs/domain/` for domain model changes; keep domain events named `NounPastTenseEvent`.
- Add/adjust tests for behavior changes.
- Validate new dependencies against `docs/governance/oss-license-policy.md`.
- For multi-step efforts, use the **plan-governor** skill: create `docs/plans/tier-<TT>/<NN>-<slug>/`
  with the four plan files, execute in verifiable increments, and keep the cross-plan `backlog.md` current.
- Keep `.agent/memory/active-context.md` thin — current state, in-flight, next priorities, risks.
  When an effort completes, use **memory-curator** to rotate narrative into `delivery-log.md` and
  durable decisions into `docs/adr/` or `decision-log.md`.
- For commits: use **commit-governor**. Conventional Commits; no AI/tool tokens in messages;
  run `scripts/pre-commit-validate.sh` before each commit; never push.
- When editing agent artifacts, follow the **agentsync** workflow: edit canonical `.agent/skills/<id>/`,
  run `dotnet agent sync`, confirm with `dotnet agent status`.

## Collaboration Contract

- Keep outputs tool-agnostic — `AGENTS.md` and canonical skills are the contract.
- Prefer repository-local docs and skills over hidden session memory.
- Write durable decisions to `docs/adr/` and `.agent/memory/decision-log.md`.

## Definition of Done

- Builds locally (`dotnet build -v q -clp:ErrorsOnly`).
- Relevant tests pass.
- Architecture constraints respected.
- New decisions recorded in `docs/adr/` or `.agent/memory/decision-log.md`.
- Commits follow `.agent/context/git-commit-policy.md` and pass repository hooks.

## Gotchas & Operational Notes

- **commit-msg hook** — Conventional Commits; rejects standalone tokens `ai|codex|claude|chatgpt|gpt|llm|agentic` (word-boundary, case-insensitive). `ai` inside a compound word (email, trail, domain) is fine.
- **Backlog gate** — a commit staging `src/` or `tests/` must also stage `docs/plans/backlog.md`. Use `[skip-progress-check] <reason>` only for genuinely plan-less commits (merges, mechanical fixups).
- **pre-commit-validate.sh** — builds the solution and runs unit tests selectively based on staged files. Keep the build fast by only touching what you changed.
- **License check is network-bound** — `check-nuget-licenses.sh` curls nuget.org; an unstable connection may give spurious `UNKNOWN` results. Re-run before concluding a real violation.
- **Windows note** — scripts target Bash (Git Bash on Windows or WSL). On Windows, run `scripts/setup-git-hooks.sh` from Git Bash. `dotnet` commands work natively on Windows.
